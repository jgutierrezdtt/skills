## Use Case: External Metadata Mapping

External metadata files map custom Fortify SAST rules to industry compliance frameworks (CWE, OWASP Top 10, PCI-DSS, NIST, ISO 27001). They do not change how rules detect vulnerabilities — they enrich findings with compliance context so that reports generated by ReportGenerator or FoD/SSC report templates can reference the appropriate standard controls.

External metadata is provided as a separate XML file (`external-metadata.xml`) that is loaded alongside the rulepack. It is not embedded within the rulepack itself.

---

### External Metadata File Structure

```xml
<?xml version="1.0" encoding="UTF-8"?>
<ExternalMetadata xmlns="xmlns://www.fortifysoftware.com/schema/externalMetadata">

  <!-- CWE mappings: link rule IDs to CWE entries -->
  <CWEMappings>
    <CWEMapping>
      <RuleID>YOUR-RULE-ID-HERE</RuleID>
      <CWE>
        <ID>89</ID>
        <Name>Improper Neutralization of Special Elements used in an SQL Command</Name>
        <URL>https://cwe.mitre.org/data/definitions/89.html</URL>
      </CWE>
    </CWEMapping>
  </CWEMappings>

  <!-- OWASP Top 10 2021 mappings -->
  <OWASPMappings version="2021">
    <OWASPMapping>
      <RuleID>YOUR-RULE-ID-HERE</RuleID>
      <Category>A03:2021-Injection</Category>
      <Name>Injection</Name>
      <URL>https://owasp.org/Top10/A03_2021-Injection/</URL>
    </OWASPMapping>
  </OWASPMappings>

  <!-- PCI-DSS v4.0 mappings -->
  <PCIDSSMappings version="4.0">
    <PCIDSSMapping>
      <RuleID>YOUR-RULE-ID-HERE</RuleID>
      <Requirement>6.2.4</Requirement>
      <Description>Software engineering techniques or other methods to prevent or mitigate common software attacks</Description>
    </PCIDSSMapping>
  </PCIDSSMappings>

</ExternalMetadata>
```

### OWASP Top 10 2021 Reference

| OWASP Category | Applicable CWEs / vulnerability types |
|----------------|---------------------------------------|
| A01: Broken Access Control | Missing authorization, IDOR, path traversal |
| A02: Cryptographic Failures | Weak crypto, hardcoded keys, cleartext storage |
| A03: Injection | SQL, command, LDAP, XSS, SSTI |
| A04: Insecure Design | Missing threat modeling patterns |
| A05: Security Misconfiguration | Debug flags, default credentials, verbose errors |
| A06: Vulnerable Components | Use of known-vulnerable library versions |
| A07: Auth Failures | Session fixation, weak passwords, missing MFA checks |
| A08: Software and Data Integrity | Insecure deserialization, unsigned actions |
| A09: Logging Failures | Poor logging practice, sensitive data in logs |

### Loading External Metadata with a Rulepack

The external metadata file must be deployed alongside the rulepack. In SSC, place the file in the same import package or in the externalmetadata directory of the Fortify installation:

```
$FORTIFY_HOME/Core/config/ExternalMetadata/
```

When running `sourceanalyzer` locally, reference both files:
```bash
sourceanalyzer -b <buildId> \
  -rules custom-rules.xml \
  -rules external-metadata.xml \
  -scan -f results.fpr
```

When importing to SSC, the metadata enrichment is applied automatically if the file is installed in the ExternalMetadata directory before the FPR is uploaded.

---

### Generating Compliance Reports from SSC

After metadata is in place, generate a compliance-oriented report from SSC using the built-in BIRT report templates. Load `skills/fortify-reporting/references/compliance-mapping.md` for the fcli commands to generate reports filtered by compliance framework.

---

### Authoring Guidelines

- Map every custom rule to at least one CWE entry. CWE is the baseline that all other frameworks build on.
- Do not fabricate CWE or OWASP category assignments. If the match is uncertain, omit the mapping and note the gap.
- OWASP and PCI-DSS mappings are at the category/requirement level — one rule may map to one or more requirements.
- Keep the external metadata file under version control alongside the rulepack. Bump the rulepack version when mappings change.
- Validate the XML schema before deployment. Malformed external metadata silently fails to load in some Fortify versions.
